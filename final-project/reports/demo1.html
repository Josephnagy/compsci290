Final Project Proposal

  • Idea
    • My idea is a web application that allows users to plan ski/snowboarding trips with friends. So users are able to add mountains (i.e. if the trip is to Killington VT, then the mountain would be Killington), flights and hotels to a list of trips that they organize and track throughout the season
    • I came up with this idea after planning snowboarding trips with friends. We would have a variety of google docs organizing our trips, a million tabs open looking at hotels, flights trail maps, etc. So I thought of this idea to create a one-stop-shop to plan snowboarding + ski trips
  • Scope
    • The main feature of the app is that users can add mountains, hotels and flights together to create trips.
    • Additionally, uses have one central place where they can browse anything from mountain trail maps, lift ticket prices, flights, and hotels all in one location
    • Mountains
      • Trail difficulty distribution (i.e. 30% beginner, 40% intermediate, 30% expert)
      • Trail Map
      • Lift ticket prices
      • Winter season dates
      • Annual snowfall
      • Current weather
    • Hotels
      • Nightly rate
      • Reviews
      • Location
      • Pictures
    • Flights
      • Cost
      • Airline
    • Trips
      • Mountains, hotels and flights for each trip
      • If time allows, I’d also like to add a “who’s bringing what” feature that allows users to track what food, supplies, etc. on each trip. The point of this would be to ensure everything important is brought and people don’t bring multiples of the same equipment
